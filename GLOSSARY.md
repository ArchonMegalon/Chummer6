# Glossary

## Player and GM terms

- **receipt**: the readable explanation of how a ruling or result was calculated
- **provenance**: where each modifier, rule path, or data point came from
- **local-first**: your session keeps working even when the network gets stupid
- **preview**: visible and usable, but not yet the final promoted public shape
- **runtime stack**: the exact rules and config combination the session is currently using
- **workbench**: the desktop and browser surface for building, inspecting, and comparing
- **play shell**: the table-facing player and GM surface
- **horizon**: a future concept intentionally kept out of the active work queue

> **Troll explains:** "preview" means you can use it, not marry it.

## Contributor tail

- **shared interface**: the package or API seam used across repo boundaries
- **split**: moving real ownership from one repo to another and deleting the old ownership
- **lockstep**: several parts moving together as one wave
- **signoff only**: visible in the program, but not a coding target
- **guide layer**: the human-facing explainer that makes the program legible without becoming a second blueprint

> **Troll explains:** if somebody says "lockstep" with too much confidence, ask which seam is actually ready.
