# Evidence Room

![Evidence Room banner](../assets/horizons/evidence-room.png "proof, but arranged by a professional instead of a caffeinated spider.")

**A grounded review room for explain and provenance output.**

_Status: Horizon only - future idea, not active build work._

_Image idea: a detective board cleaned up into a readable UI, with evidence cards, source tags, and one exhausted coffee cup that has seen too much._

## What problem does this solve?

If Chummer is going to explain itself well, humans need a place to review the evidence without drowning in raw trace noise. EVIDENCE ROOM is the idea of a dedicated review space for receipts, provenance, and approval-aware inspection.

## A real table scene

> The table wants proof, not another landfill of logs.
>
> The proof exists. The problem is that it currently looks like it was sorted by a caffeinated spider.
> **Player:** "I believe the result. I just want to see the proof without opening six debug panes."
> **GM:** "And I want to know what is verified versus inferred."
> **Chummer6:** "Evidence room loaded. Source classes grouped. Approval states visible. Trace detail collapsible."
> **Player:** "Much better."
> **GM:** "Now this feels like evidence instead of a dump truck."

## Meanwhile, Chummer is doing this

- grouping explain and provenance output into reviewable layers
- separating grounded evidence from noisier trace detail
- carrying approval state with the material
- turning proof review into a human workflow instead of a log excavation

## Why that would be great

Receipts become easier to trust because people can actually read and review them without being punished for caring.

## Why it is still a Horizon

The base evidence and provenance model still has to become fully canonical first. A review room built on drifting evidence would just make drift prettier.

## What would need to exist first

- evidence receipts
- source classification
- approvals
- preview and apply separation
